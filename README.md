# 👋 Welcome to My Learning Path

> **I don't just watch coding tutorials. I write code, make mistakes, debug problems, and build projects.**

This repository documents my hands-on journey through **programming, problem solving, frontend development, JavaScript interactivity, browser games, and Python applications**.

It began with variables, conditions, loops, arrays, functions, and patterns. From there, I started creating interfaces with **HTML and CSS**, adding behavior with **JavaScript**, experimenting with **Tailwind CSS**, and building desktop interfaces with **Python and Tkinter**.

Every folder represents a stage of learning. Every project represents a concept turned into something visible.

# ⚡ Getting Started

## 🌐 Opening Web Projects Locally

For a quick check, open `docs/index.html` in a browser. A local server is more reliable because it matches GitHub Pages behavior:

```powershell
python -m http.server 8000 --directory docs
```

Then visit `http://localhost:8000/`. Individual projects can also be opened from their `index.html` files under `docs/projects/`.

## 🔵 Running C Programs

Install a C compiler such as GCC, then compile the selected source file:

```powershell
gcc programming/c/practice/practice-problems.c -o practice-problems
./practice-problems
```

Some C files contain multiple learning exercises or intentionally commented practice sections. Review the selected file before compiling it.

## ⚙️ Running C++ Programs

```powershell
g++ programming/cpp/loops/pattern.cpp -o pattern
./pattern
```

## 🐍 Running Python Programs

Run a script with Python 3, for example:

```powershell
python programming/python/projects/password-generator/password-generator.py
```

Tkinter projects require a Python installation that includes Tk support.

## ☕ Running Java Programs

No Java file is currently present. When Java practice is added, a typical workflow will be:

```powershell
javac FileName.java
java FileName
```

# 🌍 GitHub Pages

After reviewing and pushing the changes, configure the repository:

1. Open **Settings → Pages**.
2. Set **Source** to **Deploy from a branch**.
3. Select the **main** branch.
4. Select the **/docs** folder.
5. Save and wait for GitHub to publish the site.
